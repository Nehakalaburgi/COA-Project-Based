How It Works

The 4-bit adder-subtractor circuit uses the following components and logic. The hardware itself is the same for both unsigned and signed arithmetic — only how you interpret the bits changes.

1. XOR Gates for Conditional Inversion

Each bit of input B passes through an XOR gate with the Mode signal:

B'[i] = B[i] XOR Mode - Mode = 0 (Addition): B'[i] = B[i] (B unchanged) - Mode = 1 (Subtraction): B'[i] = NOT B[i] (1's complement)

2. Full Adder Logic

Sum[i] = A[i] XOR B'[i] XOR Carry_in[i] Carry_out[i] = (A[i] AND B'[i]) OR (Carry_in[i] AND (A[i] XOR B'[i]))

3. 2's Complement for Subtraction

Step 1: Invert B → 1's complement (via XOR gates with Mode=1) Step 2: Add 1 via initial Carry_in = Mode = 1 Result: A + (2's complement of B) = A − B

4. Unsigned vs Signed Interpretation

Unsigned: treat all 4 bits as a positive value → range 0 to 15 1000 = 8, 1001 = 9, 1111 = 15 Signed (2's complement): MSB has weight −8 → range −8 to +7 1000 = −8, 1001 = −7, 1111 = −1 Formula: if MSB=1 → signed value = unsigned value − 16 The circuit hardware is IDENTICAL for both. Only the decimal interpretation of the output bits differs.

5. Overflow Detection (Signed)

Overflow = Carry_in[MSB] XOR Carry_out[MSB] - Only meaningful for signed interpretation - Indicates result exceeds signed range [−8, +7] - Unsigned arithmetic does not overflow (carry out handles it)

Binary Arithmetic Fundamentals

Binary Addition Rules

0+0=0 (carry 0) | 0+1=1 (carry 0) 1+0=1 (carry 0) | 1+1=0 (carry 1) 1+1+1=1 (carry 1) [with carry in] Example: 5 + 3 = 8 0101 (5) + 0011 (3) ------ 1000 (8 unsigned / −8 signed)

2's Complement Conversion

To find signed value when MSB = 1: signed = unsigned − 16 Example: 1011 unsigned = 11 signed = 11 − 16 = −5 To negate a number: 1. Invert all bits (1's complement) 2. Add 1 Example: −3 → 0011 → invert → 1100 → +1 → 1101 = −3 ✓

Subtraction Example (both views)

7 − 3: A = 0111 (7 unsigned / 7 signed) B = 0011 (3 unsigned / 3 signed) B inverted + carry = 1101 (2's complement of 3 = −3) 0111 + 1101 = [1]0100 (carry discarded) Result = 0100 Unsigned: 0100 = 4 ✓ (7−3=4) Signed: 0100 = 4 ✓ (7−3=4)

Overflow Example

7 + 2 (signed overflow): 0111 (7) + 0010 (2) = 1001 Unsigned: 1001 = 9 ✓ (no overflow in unsigned) Signed: 1001 = −7 ✗ (overflow! 7+2 should be 9, not −7) Overflow flag = 1 (carry_in[MSB]=0 ≠ carry_out[MSB]=1)

Conclusion

Project Summary

This Binary Calculator demonstrates the fundamental principles of digital logic circuits used in modern computers. The same 4-bit adder-subtractor hardware produces a result that can be interpreted in two ways — unsigned (0–15) or signed 2's complement (−8 to +7) — depending on the context.

Key Learning Outcomes

  • Understanding of binary number representation and arithmetic operations
  • Implementation of full adder circuits and carry propagation
  • XOR gate usage for conditional bit inversion
  • 2's complement method for subtraction in digital circuits
  • Difference between unsigned and signed interpretation of binary results
  • Overflow detection applies to signed arithmetic; carry out handles unsigned range
  • Practical application of Boolean algebra and logic gates

Real-World Applications

  • Arithmetic Logic Units (ALU): CPUs use identical hardware for signed and unsigned ops — compiler/programmer chooses interpretation
  • Digital Signal Processing: Binary arithmetic is fundamental to audio/video processing chips
  • Embedded Systems: Microcontrollers use hardware adders for efficient computation
  • Computer Architecture: Understanding these circuits is essential for processor design

Educational Value

This project serves as an excellent resource for students learning:

  • Computer Organization and Architecture
  • Digital Electronics and Logic Design
  • Embedded Systems Programming
  • Computer Engineering Fundamentals

By interacting with this simulator, you gain hands-on experience with the building blocks of modern computing!
